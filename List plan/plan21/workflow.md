# User Workflow: Multi-Tenant Marketplace

## Actor: External Brand (The Partner)

1. **Onboarding:** Brand applies for a Partner account.
2. **Verification:** Platform Admin reviews the brand's sustainability credentials and marks them as "Verified".
3. **API Integration:** Brand generates a Secure Token and connects their warehouse system.
4. **Catalog Sync:** Brand sends product data via the API. The platform automatically calculates the **Impact Index** based on the submitted material JSON.
5. **Passport Minting:** Brand "mints" a batch of Digital Product Passports via the API. Our system generates the QR tokens and queues the IPFS synchronization.

## Actor: Customer

6. **Browse Marketplace:** User visits the Shop. They can now filter by "Brand" alongside Category and Impact Tier.
7. **Cross-Brand Cart:** User adds a jacket from Brand A and sneakers from Brand B to their cart.
8. **Unified Checkout:** User completes checkout in one transaction. The system splits the order internally, calculating Carbon Gate Fees for both items based on their respective Local Hub positions.

## Actor: Platform Admin

9. **Partner Oversight:** Admin views the "Partner Pulse" in the Command Center, monitoring the total impact generated by third-party brands and managing platform commissions.
10. **Quality Control:** Admin can revoke API access or flag specific Partner passports if tampering or greenwashing is detected in factory audits.
