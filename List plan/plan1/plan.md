# Plan: MVP — EcoStore (Faceted search, Product pages, Cart/Checkout)

Status: draft
Created: 2026-05-26
Owner: Product / Planning Kernel

## Objective
Define a minimal, shippable MVP focusing on user-facing features: faceted search/filters, product detail pages with images, and a basic cart + checkout flow.

## Scope (MVP)
- Faceted search and filtering across catalog (attributes, category, price, size, color)
- Product detail pages with image gallery, variants, and basic metadata
- Cart and basic checkout (guest checkout, create order record, payment integration placeholder)

## Acceptance criteria (concise)
1. Faceted search
   - Users can search products by keyword and refine results using at least 4 facets: Category, Price range, Size, Color.
   - Search results update without full page reload (AJAX or API-driven) and show total count and pagination.

2. Product detail pages
   - Product page shows title, price, description, image gallery, available variants, and add-to-cart control.
   - High-res images load progressively; alt text present for accessibility.

3. Cart & checkout
   - Users can add/remove items from cart; cart persists in session for guests.
   - Checkout collects shipping address and email; creates an Order record with line items.
   - Payment step is pluggable; can be toggled to a sandbox mode (no live capture) for MVP.

## Minimal data models (illustrative)
- Product
  - id, sku, title, slug, description, status, price_cents, currency, category_id, created_at, updated_at

- ProductVariant
  - id, product_id, sku, price_cents, attributes(json: size,color), stock, created_at, updated_at

- Category
  - id, name, slug, parent_id

- Attribute / AttributeValue (for faceting)
  - attribute (e.g., size), value (e.g., M)

- Cart (session-backed, not persisted for MVP) or Cart (table)
  - id, session_id/user_id (nullable), expires_at

- CartItem
  - id, cart_id, product_id, variant_id, qty, unit_price_cents

- Order
  - id, order_number, email, shipping_address(json), total_cents, status, payment_payload(json), created_at

## Risks & notes
- Search performance: DB-driven faceting is acceptable for small catalogs; consider Elastic/Meilisearch if scale increases.
- Payment: keep implementation pluggable and use sandbox keys for MVP.
- Images: prefer optimized responsive images; seed includes placeholder assets for dev.
